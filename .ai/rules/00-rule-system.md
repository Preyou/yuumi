# RULE-SYSTEM

Priority: P0
Scope: `.ai/rules/*`

## Rule
- 所有规则文件必须使用统一结构：`Priority`、`Scope`、`Rule`、`Non-Goals`、`Exceptions`、`Checks`。
- 规则冲突按优先级处理：`P0 > P1 > P2`。
- 同优先级规则冲突时，优先采用 `Scope` 更窄的规则。
- 任何例外都必须记录在变更 diff 或 PR description 中。
- 全仓代码默认采用 Fail-Fast：禁止为了“看起来稳定”而引入过度异常兜底、静默降级、吞错继续执行。
- 底层错误应优先抛出并中断执行，尽早暴露问题并修复根因，而不是以兜底逻辑掩盖问题。
- 仅在明确边界层允许处理错误（如用户提示、重试、外部依赖隔离），且必须保留失败信号，不得伪造成功结果。

## Non-Goals
- 本文件不定义具体编码风格细节。
- 本文件不定义业务行为本身。

## Exceptions
- 无。

## Checks
- 评审新增规则文件时，确认必需章节完整存在。
- 对只有口号、没有可验证检查项的规则，直接拒绝合入。
- 抽查关键改动路径时，检查是否存在空 `catch`、吞错后继续成功流或以默认值掩盖错误的逻辑。
- 检查边界层错误处理是否仍保留失败信号（抛错、失败态返回或明确失败日志）。
